# QFinanceData 前端看板产品文档

## 1. 文档边界

本文档描述 QFinanceData 前端看板的产品目标、用户场景、页面结构、功能范围和交互流程。

本文档不描述具体技术选型、系统架构、接口设计、数据库结构或工程实现细节。相关内容见 [technical-design.md](technical-design.md)。

## 2. 产品概述

QFinanceData 是一个面向个人投研和数据研究的金融数据抓取与看板工具。用户可以管理自选标的，查看行情走势、关键指标、财务摘要、数据新鲜度和抓取任务状态。

第一版产品优先满足本地研究和数据准备场景，不定位为实时交易终端。

## 3. 产品目标

### 3.1 核心目标

- 管理自选标的列表，支持美股、港股、ETF 和指数。
- 查看自选标的的最新价格、涨跌幅、成交量和更新时间。
- 查看单个标的的价格走势、成交量、关键指标、财务摘要和分红拆股事件。
- 手动触发数据更新，并查看更新进度、成功状态和失败原因。
- 清楚标识数据是否新鲜、过期、缺失或抓取失败。

### 3.2 非目标

- 第一版不做实时交易下单。
- 第一版不做完整投资组合记账。
- 第一版不做高频行情或毫秒级实时数据。
- 第一版不做复杂权限系统。
- 第一版不做完整回测系统。

## 4. 目标用户

### 4.1 主要用户

- 个人投资者：希望集中查看关注标的的走势、指标和数据状态。
- 量化研究者：需要批量维护本地行情数据，并快速确认数据是否完整。
- 财务分析用户：希望快速查看标的基础财务数据和公司摘要。

### 4.2 典型使用场景

- 每天收盘后更新一批自选标的的数据。
- 打开看板查看哪些标的上涨、下跌或数据过期。
- 进入单个标的详情页查看价格走势、成交量和基础指标。
- 发现更新失败后，在任务页查看失败原因并重新更新。
- 将本地数据用于后续 Notebook、分析脚本或研究流程。

## 6. 全局布局

### 6.1 桌面端布局

```text
┌──────────────────────────────────────────────────────────────┐
│ QFinanceData        Search ticker...       1D 1M 1Y MAX  ⟳  │
├───────────────┬──────────────────────────────────────────────┤
│ Dashboard     │                                              │
│ Watchlist     │                                              │
│ Jobs          │              当前页面主内容                  │
│ Settings      │                                              │
└───────────────┴──────────────────────────────────────────────┘
```

### 6.2 全局组件

- 左侧导航：固定显示核心页面入口。
- 顶部搜索：支持输入 ticker 或公司名称。
- 时间区间控件：`1D`、`1M`、`3M`、`1Y`、`5Y`、`MAX`。
- 刷新按钮：对当前页面上下文触发数据更新。
- 数据状态提示：展示当前页面数据是否新鲜、过期、缺失或失败。

### 6.3 移动端策略

第一版优先桌面端。移动端只保证可查看，不优先优化复杂交互。

- 左侧导航折叠为顶部或底部菜单。
- 图表区域纵向排列。
- 表格切换为列表卡片。
- 更新操作保持容易触达。

## 7. 页面规划

### 7.1 Dashboard 总览页

#### 页面目标

让用户在第一屏快速判断市场概况、自选标的表现、数据是否新鲜、更新任务是否异常。

#### 页面草图

```text
┌──────────────────────────────────────────────────────────────┐
│ Market Overview                         Last update 16:30    │
├──────────────────────────────────────────────────────────────┤
│ SPY +0.42%   QQQ +0.71%   ^GSPC +0.38%   ^IXIC +0.65%        │
├───────────────────────┬──────────────────────────────────────┤
│ Watchlist             │ Watchlist Trend                      │
│ AAPL     213.40 +1.2% │                                      │
│ MSFT     487.20 -0.4% │      return line chart               │
│ NVDA     154.80 +2.8% │                                      │
│ 0700.HK  392.00 +0.9% │                                      │
├───────────────────────┼──────────────────────────────────────┤
│ Top Movers            │ Data Freshness                       │
│ Gainers / Losers      │ Fresh 21  Stale 3  Failed 1          │
├───────────────────────┴──────────────────────────────────────┤
│ Recent Jobs: prices success / fundamentals failed             │
└──────────────────────────────────────────────────────────────┘
```

#### 模块说明

| 模块 | 说明 |
| --- | --- |
| 市场摘要 | 展示核心指数或 ETF 的最新涨跌幅。 |
| 自选标的概览 | 展示 ticker、最新价、涨跌幅、更新时间和状态。 |
| 自选走势 | 展示自选列表等权收益率曲线或选中标的走势。 |
| 涨跌排行 | 展示自选标的中的涨幅榜、跌幅榜。 |
| 数据新鲜度 | 汇总 fresh、stale、missing、failed 状态。 |
| 最近任务 | 展示最近几次更新任务的类型、状态和错误摘要。 |

#### 关键交互

- 点击自选标的进入详情页。
- 点击涨跌排行中的标的进入详情页。
- 点击数据新鲜度模块跳转至任务页。
- 点击最近失败任务跳转至 Jobs 页面并定位该任务。

### 7.2 Symbol Detail 标的详情页

提供单个标的的价格走势、成交量、关键指标、财务摘要和数据状态。

```text
├───────────────────────┬──────────────────────────────────────┤
```

#### 模块说明

| 模块 | 说明 |
| --- | --- |
| 标题区 | ticker、公司名称、最新价、涨跌幅、刷新和导出操作。 |
| 价格图表 | 支持 K 线、复权收盘价、移动均线。 |
| 成交量 | 与价格图同时间范围联动。 |
| 关键指标 | 展示市值、PE、PB、股息率、52 周高低点等。 |

#### 关键交互

- 切换时间范围后，价格图、成交量和事件标记同步更新。
- 点击 `Refresh` 触发当前标的的数据更新。
- 点击 `Export` 导出当前标的行情或基础数据。
- 图表 hover 时显示 OHLC、成交量、复权价格和事件标记。

### 7.3 Watchlist 标的管理页

#### 页面目标

管理用户关注的 ticker、分组和数据状态。

```text

│ Watchlist                              Add ticker: [____] +   │
├──────────────────────────────────────────────────────────────┤

│ Symbol    Name          Type     Currency   Last Update Status│

│ 0700.HK   Tencent       Equity   HKD        2026-06-21  Stale │
│ SPY       S&P 500 ETF   ETF      USD        2026-06-22  Fresh │
└──────────────────────────────────────────────────────────────┘
```

#### 模块说明

| 模块 | 说明 |
| --- | --- |
| 添加 ticker | 输入 ticker 并加入默认分组。 |
| 分组筛选 | 按市场、资产类型或用户自定义分组筛选。 |

| 批量导入 | 支持一行一个 ticker。 |

- 添加 ticker 后显示在 Watchlist，并提示用户是否立即更新数据。
- 删除 ticker 时只移出 Watchlist，不默认删除历史数据。
- 支持批量选择标的并触发数据更新。
- 点击表格行进入标的详情页。

#### 页面目标

管理和观察数据更新任务，让用户知道任务是否成功、失败在哪里、是否需要重试。

```text
│ History                                                      │
│ Time        Type          Symbols   Status     Error          │
│ 16:30       prices        24        success    -              │
│ 16:10       fundamentals  24        failed     timeout AAPL   │
│ 15:58       actions       24        success    -              │
└──────────────────────────────────────────────────────────────┘
```

#### 模块说明

| 模块 | 说明 |
| --- | --- |
| 手动更新 | 支持价格、基础信息、财务摘要、分红拆股等数据类型。 |
| 当前队列 | 展示等待中和运行中的任务。 |
| 进度条 | 展示标的级别的完成数量。 |
| 历史记录 | 展示任务时间、类型、标的数量、状态和错误摘要。 |
| 错误详情 | 展示失败标的、错误类型、错误消息和重试入口。 |

#### 关键交互

- 用户可以对失败任务执行重试。
- 用户可以按任务类型、状态、时间过滤历史记录。
- 点击任务行进入任务详情。
- 运行中任务需要自动刷新状态。

### 7.5 Settings 设置页

#### 页面目标

管理默认更新行为和显示偏好。第一版设置页保持轻量。

#### 第一版字段

| 设置项 | 默认值 | 说明 |
| --- | --- | --- |
| 默认时间范围 | `1Y` | 图表和行情查询的默认范围。 |
| 默认更新起点 | `2015-01-01` | 首次更新历史数据的默认起始日期。 |
| 涨跌颜色 | 美股习惯 | 绿色上涨、红色下跌；可切换 A 股习惯。 |
| 数据过期阈值 | 2 个交易日 | 超过阈值则标记为 stale。 |
| 默认价格口径 | 复权价格 | 控制图表优先展示复权或未复权价格。 |

## 8. 用户可见状态

### 8.1 数据状态

| 状态 | 含义 |
| --- | --- |
| `fresh` | 数据在设定阈值内更新。 |
| `stale` | 数据超过阈值未更新。 |
| `missing` | 本地没有该类数据。 |
| `failed` | 最近一次更新失败。 |
| `partial` | 部分标的或部分数据类型更新失败。 |

### 8.2 任务状态

| 状态 | 含义 |
| --- | --- |
| `queued` | 已创建，等待执行。 |
| `running` | 正在执行。 |
| `success` | 全部成功。 |
| `partial_success` | 部分成功，部分失败。 |
| `failed` | 全部失败或任务级错误。 |
| `cancelled` | 用户取消。 |

## 9. 关键用户流程

### 9.1 添加新标的

```text
输入 ticker
  -> 系统校验 ticker
  -> 显示标的名称、市场和币种
  -> 用户确认加入 Watchlist
  -> 用户选择是否立即更新数据
```

### 9.2 手动更新价格

```text
点击 Update Prices
  -> 选择更新范围
  -> 系统创建更新任务
  -> Jobs 页面展示进度
  -> 完成后刷新 Dashboard 和详情页
```

### 9.3 查看失败任务

```text
进入 Jobs
  -> 筛选 failed / partial_success
  -> 点击任务详情
  -> 查看失败标的和错误消息
  -> 点击 Retry
  -> 只重试失败标的
```

- Dashboard 总览页。
- Symbol Detail 标的详情页。

- Jobs 更新任务页。

## 11. 产品设计原则

- 信息密度优先，避免营销页式大面积装饰。
- 图表是主角，卡片只用于承载具体信息块。
- 全局时间范围和标的搜索始终容易访问。
- 数据状态必须显性展示，不能隐藏在日志里。
- 错误提示要指向可操作动作，例如重试、查看详情、重新更新。
- 上涨/下跌颜色需要可配置，适配不同市场习惯。

## 12. 产品风险与约束

| 风险 | 影响 | 产品侧应对 |
| --- | --- | --- |
| 数据源可能不稳定 | 用户看到缺失或过期数据 | 明确展示 fresh、stale、missing、failed 状态。 |
| 不同市场更新时间不同 | 用户误判数据是否过期 | 允许设置数据过期阈值，并展示最后更新时间。 |
| 财务字段在不同标的间不一致 | 用户难以比较 | 第一版只展示少量稳定摘要字段。 |
| 批量更新耗时较长 | 用户不知道是否卡住 | Jobs 页面展示进度和当前任务。 |
| ticker 格式不熟悉 | 用户添加失败 | 添加时展示校验结果和失败原因。 |

## 13. 产品迭代计划

### Phase 1: 看板原型

- 使用模拟数据完成 Dashboard、Symbol Detail、Watchlist、Jobs 和 Settings。
- 验证布局、页面信息密度和主要交互。

### Phase 2: 数据更新 MVP

- 支持自选标的真实增删。
- 支持手动更新日线价格。
- Dashboard 和详情页展示真实行情数据。
- Jobs 展示真实任务状态。

### Phase 3: 数据增强

- 增加基础信息、财务摘要、分红拆股。
- 增加数据新鲜度和异常提示。
- 增加导出入口。

### Phase 4: 自动化与扩展

- 增加定时更新。
- 增加失败重试入口。
- 增加更多图表和指标。
- 根据需要增加通知、权限和部署能力。
